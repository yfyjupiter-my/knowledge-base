Low-fidelity, monochrome sketches of the three core flows from the PRD: signing in with a restricted Google account, browsing/searching the canonical document set, and viewing a versioned document with compliance sign-off.

Screen 3 — Document Detail / Editor (versioning + compliance sign-off)

1280×800 desktop
  1. 1 — navbar (shared shell)
  2. 2 — breadcrumb: Documents / Category / Title
  3. 3 — document title + category & status badges
  4. 4 — metadata bar: owner, last reviewed (overdue in red), review interval, region, tags
  5. 5 — toolbar: Edit, view history, attach file, request approval
  6. 6 — rich-text body with numbered sections and inline table/attachment placeholders
  7. 7 — approver sign-off panel — blocks Draft → Published for Compliance docs
  8. 8 — version history list with diff links (append-only, nothing hard-deleted)
  9. 9 — audit log excerpt (who/what/when) with link to full trail
Assumptions: viewer has Approver role (sign-off box visible); doc is Compliance category so approval gate applies — Technical Guide/System Config docs would omit box 7 and publish directly.
